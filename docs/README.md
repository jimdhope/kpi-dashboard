# KPI Quest Documentation

Welcome to the documentation for **KPI Quest (v3.5.2)**, a comprehensive performance management, competition tracking, and engagement platform built with Next.js, Prisma, and PostgreSQL.

This documentation is divided into multiple parts, tailored to different audiences:

## 📚 Table of Contents

1. [**Introduction & Core Concepts**](./introduction.md)
   - High-level overview of the KPI Quest platform.
   - Core pillars: Performance Tracking, Gamified Competitions, Knowledge Sharing, and Team Automation.
   - Core glossary of terms.

2. [**User & Agent Guide**](./user-guide.md)
   - A friendly guide for Agents, Team Leaders, and general users.
   - How to track daily KPIs, view performance breakdowns, and participate in competitions.
   - Operating the Directory, Knowledge Base, built-in Tools, and Mini-Games.

3. [**Administrator & Configuration Guide**](./admin-guide.md)
   - Configuration of Campaigns, Pods, and Users.
   - Setting up MS Teams webhooks, automation rules, templates, and scheduled postings.
   - Score target configurations for webhook hashtag scoring.
   - Backup, restore, and data importing workflows.
   - Troubleshooting common administrative tasks.

4. [**Developer & Technical Reference**](./developer-reference.md)
   - Deep technical reference for software engineers maintaining or extending KPI Quest.
   - Architecture & Directory Layout (Next.js route groups, API structure).
   - Detailed Prisma Database Schema analysis.
   - Custom Session Authentication & Role-Based Access Control (RBAC).
   - Core service layers and background jobs worker.
   - Webhook automation engine & adaptive card rendering with Resvg.
   - Developer commands, building, deployment, and testing.

5. [**Local Performance and Memory Baseline**](./performance-baseline.md)
   - Repeatable build, bundle-analysis, instrumentation, and memory diagnostics.
